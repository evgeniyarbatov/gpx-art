## End-to-end flow

```
SOURCE_DIR (your GPX library)
        ‚îÇ
        ‚ñº
  select subset ‚îÄ‚îÄ random (make random) or DTW diversity (make dtwselect)
        ‚îÇ
        ‚ñº
     gpx/          working input set
        ‚îÇ
        ‚ñº
  gpx-art.py       for each track √ó each style
        ‚îÇ
        ‚îú‚îÄ style function ‚Üí matplotlib figure
        ‚îú‚îÄ (optional) extract style source ‚Üí gist ‚Üí QR overlay
        ‚ñº
   images/         <style>-<track>.png
```

1. Collect GPX files into `gpx/` (random sample or DTW selection from a source directory).
2. Optionally preview tracks with `plot-gpx.py`.
3. Render each GPX file with every registered style in `gpx-art.py`.
4. Optionally publish/reuse a GitHub Gist of the style source and embed a QR code (skipped with `--no-qr`).
5. Write PNGs to `images/`.

## Components

| Path | Role |
|---|---|
| `scripts/gpx-art.py` | Main generator: style registry, rendering, optional QR |
| `scripts/dtw-select.py` | Diverse GPX selection via FastDTW |
| `scripts/plot-gpx.py` | Visual grid preview of tracks in `gpx/` |
| `scripts/gist.py` | GitHub Gist create/reuse + SQLite cache |
| `scripts/utils.py` | Shared GPX listing and parse helpers |

| `images/` | Generated PNG outputs |
| `gists.db` | Local cache of style name ‚Üí gist URL (gitignored) |

## Style system

Styles are plain Python functions registered with a decorator:

```python
@style("scaffold")
def scaffold(lons, lats):
    # draw with matplotlib ‚Ä¶
    return fig, bg_color
```

- `STYLES` maps name ‚Üí callable.
- Each style receives longitude/latitude arrays and returns a figure plus background color.
- `create_art` looks up the style, draws the track, optionally adds a QR, and saves the PNG.
- New styles are added by defining another `@style(...)` function in `gpx-art.py`; no central switch statement.

Path helpers (`essence_path`, `flow_path`, `gap_mask`, ink strokes, palettes, etc.) are shared utilities used by many styles.

## Provenance (QR + Gist)

When QR is enabled:

1. `extract_style_source` parses `gpx-art.py` with the AST and pulls the full `@style` function for that name.
2. `gist.get_gist_url` hashes the source and checks `gists.db`.
3. Unchanged hash ‚Üí reuse cached URL; changed or new ‚Üí create a public Gist via the GitHub API.
4. A QR code encoding that URL is placed in the bottom-right of the figure.

With `--no-qr`, Gist and QR steps are skipped entirely ‚Äî no `GITHUB_TOKEN` required.

## Track selection

**Random** (`make random`): `find` + `shuf` + `cp` ‚Äî simple sample from `SOURCE_DIR`.

**DTW** (`make dtwselect` / `dtw-select.py`):

1. Parse all GPX files in the source directory.
2. Drop tracks shorter than 10 km.
3. Downsample and normalize trajectories.
4. Greedily pick a diverse set maximizing FastDTW distance between selected tracks.
5. Copy winners into `gpx/`.

## Data layout

- `SOURCE_DIR` (default `./source-gpx`) is the external library; the repo does not ship large source collections.
- `gpx/` and `images/` are working directories regenerated by Make targets.
- Secrets live in `.env` (`GITHUB_TOKEN`); both `.env` and `gists.db` are gitignored.
